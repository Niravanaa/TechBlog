Course and Book Progress

I've been on an adventure, diving deep into the world of Git and GitHub. Let's take a look at what I've learned so far:

Thursday

In "The Git and GitHub Bootcamp" by Colt Steele, I learned about handling merge conflicts, creating and switching branches, and using VSCode for efficient collaboration. It's essential to understand how to resolve conflicts and create clean merges in collaborative projects.

In "Apprenticeship Patterns" by Dave Hoover and Adewale Oshineye, I discovered the importance of finding like-minded peers in the software development world. Learning and growing with others who share your passion can be incredibly rewarding.

Saturday

On this day, my Git and GitHub journey continued. I delved into topics like reading diffs, viewing changes in the working directory, and mastering Git stashes. Git stash is a valuable tool for temporarily saving changes while switching between branches.

The "Apprenticeship Patterns" book introduced the concepts of practice and breakable toys. Continuous practice and experimentation are crucial for personal growth as a developer.

Sunday

Sunday was all about Git stash, applying stashed changes, working with multiple stashes, and understanding Git stash lists. Git stash is a versatile feature that helps manage changes effectively during branch switching.

The "Apprenticeship Patterns" book emphasized the importance of using feedback loops, learning from failures, maintaining a reading list, and constant learning.

Monday

I learned how to check out old commits, re-attach a detached HEAD, reference commits relative to HEAD, and discard changes with Git checkout. These Git commands are essential for managing project history.

In "Apprenticeship Patterns," I explored the ideas of recording what you learn and sharing your knowledge with others. Teaching and learning from peers is a valuable part of the developer's journey.

Upcoming Projects

While I didn't get to work on a project this week, I'm excited about diving into an SQL online course and starting a Spring Boot project with a website application/dashboard. Let's briefly explore what Spring Boot and SQL are:

Spring Boot

Spring Boot is a Java-based framework that simplifies the development of stand-alone, production-grade Spring-based applications. It provides a wide range of features, including an embedded web server, easy configuration, and production-ready metrics. Spring Boot is perfect for building web applications and microservices with speed and ease.

SQL (Structured Query Language)

SQL is a powerful language for managing and querying relational databases. It allows you to create, modify, and retrieve data from databases. In my Spring Boot project, I will try working with databases using SQL to store and retrieve information efficiently. I will keep an eye out for additional derivatives like MySQL, PostgreSQL, or SQLite.
